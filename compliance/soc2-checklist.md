# SOC 2 Readiness Checklist

Use this when the product stores customer data, supports team access, or makes security claims to buyers.

## Security Foundations

- Access control with role-based permissions
- SSO support or documented path to SSO
- Secrets stored outside markdown memory
- Audit logs for privileged actions
- Change management process
- Incident response owner and runbook

## Platform Controls

- Production and development separation
- Backup and restore procedure
- Dependency and vulnerability review
- Logging and alerting coverage
- Vendor inventory for third-party processors

## Evidence Artifacts

- Architecture and data-flow summary
- Access review log
- Incident register
- Change log or release record
- Risk register

## Gate

Do not claim SOC 2 readiness unless each control has an owner and evidence path.
