# System Rules

- Load this file at the start of every session.
- Prefer markdown knowledge over server-side workflow logic.
- Use MCP only for actions that require live side effects or secret-backed access.
- Do not store secrets in memory.
- Derive a minimum viable system before proposing architecture.
- Stop for approval after the roadmap.
- Keep summary files short and current.
- Use the timeline for recovery, not for permanent strategy.
- Archive stale knowledge instead of deleting history.
- Use git as a ledger when version control is available.
