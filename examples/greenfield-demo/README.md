# Greenfield Demo

This example shows the product with no repo path at all.

Brief:

> Build a support inbox product for customer success teams. Capture inbound requests, route them to the right owner, and keep follow-ups visible until resolution.

## What the product generated

- [TASK_BREAKDOWN.md](./TASK_BREAKDOWN.md): concrete smaller tasks derived from the brief
- [task-primary-flow.md](./task-primary-flow.md): one native task agent generated from that breakdown

## What this demonstrates

- the product does not need an existing codebase to create structure
- it can break a new product into smaller, simpler tasks
- it can emit task agents the flagship coding tools can use directly once implementation begins
